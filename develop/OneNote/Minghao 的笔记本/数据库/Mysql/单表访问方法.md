2. Const

通过主键定位一条记录或者唯一二级索引与常数进行等值比较，只能定位到一条数据

4. Ref

某个二级索引与常数进行等值比较，可能会扫出多条数据，不需要完全匹配，最左匹配即可，但是需要是等值比较

6. Ref_or_null

不仅扫出二级索引列中等于某个常数的记录，还想把该列中值为NULL的部分也扫出来  
非等值比较

8. Range

使用索引执行查询时，对应的扫描区间为若干个单点扫描区间或者范围扫描区间，仅包含一个扫描区间的访问方法不能称为range访问方法，扫描区间为（-∞，+∞）的也不能称为range访问方法

10. Index

所有的查询列都在index中，因为index比聚蔟索引少字段少，所以访问起来会更快，如果orderby主键，也会被认为是使用index访问方法

12. All

全表
  
每次从二级索引中读取到一条记录之后都会立刻回表去聚簇索引中查找值，这样会产生非常多的随机IO，但是有一种优化手段叫做MRR，multi-range read,多范围读取的措施，可以攒一波二级索引，将主键排好序后再统一执行回表操作，但是这种优化开启的条件比较苛刻的。、
 
索引合并

1. Intersection索引合并，先取两个二级索引主键结果的交集，每取到一条交集后立刻回表，要求使用一个索引时二级索引过滤出来的结果有序
2. Union索引合并，取两个二级索引主键的并集，每取到一条并集后立刻回表，要求使用一个索引时二级索引过滤出来的结果有序
